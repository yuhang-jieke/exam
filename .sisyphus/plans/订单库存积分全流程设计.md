## 一、订单全流程

### 1.1 订单状态机

```mermaid
stateDiagram-v2
    [*] --> CREATING: 开始创建
    CREATING --> PENDING_PAYMENT: 提交订单
    PENDING_PAYMENT --> PAID: 支付成功
    PENDING_PAYMENT --> CANCELLED: 超时未支付
    PAID --> PENDING_SHIPMENT: 支付确认
    PENDING_SHIPMENT --> SHIPPED: 商家发货
    SHIPPED --> DELIVERED: 用户收货
    DELIVERED --> COMPLETED: 确认收货/7 天自动
    COMPLETED --> [*]: 流程结束
    
    %% 取消和退款路径
    CANCELLED --> [*]: 取消完成
    PAID --> REFUNDING: 申请退款
    PENDING_SHIPMENT --> REFUNDING: 申请退款
    SHIPPED --> REFUNDING: 申请退货
    REFUNDING --> REFUNDED: 退款完成
    REFUNDED --> [*]: 流程结束
```

**状态说明**:
- `CREATING`: 订单创建中（购物车）
- `PENDING_PAYMENT`: 待支付（库存已锁定）
- `PAID`: 已支付（待发货）
- `PENDING_SHIPMENT`: 待发货
- `SHIPPED`: 已发货
- `DELIVERED`: 已收货
- `COMPLETED`: 已完成
- `CANCELLED`: 已取消
- `REFUNDING`: 退款中
- `REFUNDED`: 已退款

### 1.2 订单创建时序图

```mermaid

    DB-->>Order: 订单创建成功

    MQ-->>Order: 消息已入队
    
    Order-->>API: 返回订单号
    API-->>User: 订单创建成功，去支付
    
    Note over Order,MQ: 异步消息处理
    MQ->>MQ: 订单消息 (goods.queue)
    
    par 并行处理
        Order->>MQ: 发送库存扣减消息
        MQ->>Inventory: 消费扣减消息
        Inventory->>DB: 扣减库存
    end
```

## 二、库存全流程

### 2.1 库存状态机

```mermaid
stateDiagram-v2
    [*] --> AVAILABLE: 可用库存
    
    AVAILABLE --> RESERVED: 下单预占
    RESERVED --> AVAILABLE: 取消订单释放
    RESERVED --> DEDUCTED: 支付成功扣减
    
    DEDUCTED --> RESTORED: 退货入库
    RESTORED --> AVAILABLE: 上架销售
    
    AVAILABLE --> LOCKED: 库存盘点锁定
    LOCKED --> AVAILABLE: 盘点完成解锁
```

**状态说明**:
- `AVAILABLE`: 可用库存（可销售）
- `RESERVED`: 已预占（订单待支付）
- `DEDUCTED`: 已扣减（订单已支付）
- `LOCKED`: 锁定中（盘点/调拨）
- `RESTORED`: 已恢复（退货）

### 2.2 库存扣减时序图

```mermaid
sequenceDiagram
    autonumber
    participant Order as 订单服务
    participant MQ as RabbitMQ
    participant Inventory as 库存服务
    participant DB as 数据库
    participant Redis as Redis
    
    Order->>MQ: 发送扣减库存消息
    
    MQ->>Inventory: 消费消息
    
    Inventory->>Redis: 检查防重 Key
    Redis-->>Inventory: 未处理过
    
    Inventory->>DB: 开启事务
    Inventory->>DB: 查询库存 (SELECT ... FOR UPDATE)
    DB-->>Inventory: 当前库存数量
    
    Inventory->>DB: 扣减库存 (UPDATE ... SET stock=stock-?)
    Inventory->>DB: 记录库存流水
    
    Inventory->>Redis: 设置防重 Key (24 小时)
    
    par 库存预警
        Inventory->>Inventory: 检查库存是否低于阈值
        Inventory->>MQ: 发送补货预警消息
    end
    
    DB-->>Inventory: 提交事务
    Inventory-->>MQ: 处理成功
```

### 2.5 库存核心表结构

```sql
-- 库存表
CREATE TABLE inventory (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    product_id BIGINT NOT NULL,
    warehouse_id BIGINT NOT NULL,
    
    -- 库存数量
    stock INT NOT NULL DEFAULT 0 COMMENT '可用库存',
    locked_stock INT NOT NULL DEFAULT 0 COMMENT '锁定库存',

    -- 库存水位
    warn_stock INT NOT NULL DEFAULT 10 COMMENT '预警库存',
    max_stock INT NOT NULL DEFAULT 1000 COMMENT '最大库存',

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    UNIQUE KEY uk_product_warehouse (product_id, warehouse_id),

-- 库存流水表
CREATE TABLE inventory_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    product_id BIGINT NOT NULL,
    warehouse_id BIGINT NOT NULL,
    
    -- 变动信息

    before_qty INT NOT NULL COMMENT '变动前数量',
    after_qty INT NOT NULL COMMENT '变动后数量',
    
    -- 关联信息
    order_id VARCHAR(32) COMMENT '订单 ID',
    biz_type VARCHAR(50) COMMENT '业务类型',
    biz_id VARCHAR(100) COMMENT '业务 ID',
    
    remark VARCHAR(255) COMMENT '备注',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_product_id (product_id),
    INDEX idx_order_id (order_id),

    order_id VARCHAR(32) NOT NULL,

    status TINYINT NOT NULL DEFAULT 0 COMMENT '状态',
    expire_time TIMESTAMP NOT NULL COMMENT '过期时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE KEY uk_order_product (order_id, product_id),
    INDEX idx_expire_time (expire_time)
);
```

## 三、积分全流程

### 3.1 积分状态机

```mermaid
stateDiagram-v2
    [*] --> PENDING: 待结算
    
    PENDING --> AVAILABLE: 订单完成
    PENDING --> CANCELLED: 订单取消
    
    AVAILABLE --> FROZEN: 积分使用冻结
    FROZEN --> AVAILABLE: 使用失败解冻
    FROZEN --> USED: 使用成功扣减
    
    AVAILABLE --> EXPIRED: 积分过期
    USED --> [*]: 已使用
    EXPIRED --> [*]: 已过期
```

**状态说明**:
- `PENDING`: 待结算（订单待完成）
- `AVAILABLE`: 可用积分
- `FROZEN`: 已冻结（积分支付中）
- `USED`: 已使用
- `CANCELLED`: 已取消
- `EXPIRED`: 已过期

### 3.2 积分获取时序图

```mermaid
sequenceDiagram
    autonumber

    DB-->>Points: 规则信息
```

```mermaid
    Order->>MQ: 发送订单支付消息
    
    par 扣减库存
        MQ->>Inventory: 消费消息
        Inventory->>DB: 确认扣减
        Inventory->>Redis: 设置防重
    end
    
    par 扣减积分
        MQ->>Points: 消费消息
        Points->>DB: 确认扣减
        Points->>Redis: 设置防重
    end
    
    par 赠送积分
        MQ->>Points: 消费消息
        Points->>DB: 计算并赠送
        Points->>Redis: 设置防重
    end
```
