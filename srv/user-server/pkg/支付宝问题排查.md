# 支付宝支付返回空 URL 问题排查指南

## 问题描述

调用 `AliPay` 方法返回 `{ "url": "" }`，即支付 URL 为空字符串。

## 可能的原因和解决方案

### 原因 1：配置未正确加载 ✅ 已修复

**问题**：`config.GlobalConf.Pay` 为空或未初始化

**检查方法**：
```go
if config.GlobalConf == nil {
    log.Println("[AliPay] 错误：全局配置未初始化")
    return ""
}
```

**解决方案**：
确保在 `main.go` 中正确调用了初始化：
```go
inits.ConfigInit()  // 必须先加载配置
```

### 原因 2：PrivateKey 为空 ✅ 已修复

**问题**：配置文件中 `PrivateKey` 字段为空或格式不正确

**检查方法**：
```go
if alipayinit.PrivateKey == "" {
    log.Println("[AliPay] 错误：PrivateKey 为空")
    return ""
}
```

**解决方案**：
检查 `config.yaml` 中的 `AliPay.PrivateKey` 配置：
```yaml
AliPay:
  PrivateKey: "MIIEowIBAAKCAQEA..."  # 必须是完整的 RSA 私钥
```

### 原因 3：AppId 为空 ✅ 已修复

**问题**：配置文件中 `AppId` 为空

**检查方法**：
```go
if alipayinit.AppId == "" {
    log.Println("[AliPay] 错误：AppId 为空")
    return ""
}
```

**解决方案**：
确保 `config.yaml` 中配置了正确的支付宝 AppId：
```yaml
AliPay:
  AppId: "9021000157678353"  # 替换为你的实际 AppId
```

### 原因 4：支付宝 SDK 初始化失败

**问题**：私钥格式不正确或 SDK 初始化失败

**日志输出**：
```
[AliPay] 创建客户端失败：xxx error
```

**解决方案**：
1. 确认私钥是 RSA 私钥（2048 位）
2. 确认私钥不包含头部和尾部（如 `-----BEGIN RSA PRIVATE KEY-----`）
3. 使用支付宝官方工具重新生成密钥

### 原因 5：TradeWapPay 创建 URL 失败

**问题**：调用 `client.TradeWapPay(p)` 返回错误

**日志输出**：
```
[AliPay] 创建支付 URL 失败：xxx error
```

**解决方案**：
1. 检查 `NotifyURL` 和 `ReturnURL` 是否配置
2. 确认 URL 是有效的 HTTP/HTTPS 地址
3. 确保订单号 `OutTradeNo` 唯一

### 原因 6：订单查询失败

**问题**：在 `handler/server.go` 中查询订单失败

**检查方法**：
```go
id, err := order.FindId(config.DB, in.Id)
if err != nil {
    return nil, errors.New("查询失败")
}
```

**解决方案**：
1. 确保数据库中有该订单
2. 检查 `Order` 模型的 `FindId` 方法实现
3. 确认订单 ID 存在

## 完整的日志输出（正常情况）

## 调试步骤

### 步骤 1：检查配置加载

在 `AliPay` 调用前添加日志：
```go
log.Printf("[DEBUG] GlobalConf=%+v", config.GlobalConf)
log.Printf("[DEBUG] Pay=%+v", config.GlobalConf.Pay)
```

### 步骤 2：检查配置文件格式

确保 `config.yaml` 格式正确（注意缩进）：
```yaml
AliPay:
  PrivateKey: "MIIEow..."
  AppId: "9021000157678353"
  NotifyURL: "http://xxx/notify/pay"
  ReturnURL: "http://xxx/callback"
```

### 步骤 3：验证私钥格式

使用 OpenSSL 验证私钥：
```bash
# 将私钥保存到文件
echo "MIIEowIBAAKCAQEA..." > private_key.pem

# 验证私钥格式
openssl rsa -in private_key.pem -check -noout
```

应该输出：`RSA key ok`

### 步骤 4：测试支付宝连接

使用支付宝 SDK 的测试功能：
```go
client, err := alipay.New(appId, privateKey, false)
if err != nil {
    log.Fatalf("SDK 初始化失败：%v", err)
}
log.Println("支付宝 SDK 初始化成功")
```

## 配置文件检查清单

- [ ] `config.yaml` 中有 `AliPay` 配置段
- [ ] `PrivateKey` 不为空且格式正确
- [ ] `AppId` 是有效的支付宝应用 ID
- [ ] `NotifyURL` 是可访问的公网地址
- [ ] `ReturnURL` 是可访问的公网地址
- [ ] 配置文件编码为 UTF-8
- [ ] YAML 缩进正确（使用空格，不是 Tab）

## 常见错误及解决方案

### 错误 1：`private key is invalid`

**原因**：私钥格式不正确

**解决**：
1. 确保私钥不包含 `-----BEGIN RSA PRIVATE KEY-----` 和 `-----END RSA PRIVATE KEY-----`
2. 确保私钥是单行或正确格式化的多行字符串

### 错误 2：`app id not found`

**原因**：AppId 配置错误或应用未审核通过

**解决**：
1. 检查 AppId 是否正确
2. 登录支付宝开放平台确认应用状态

### 错误 3：`notify_url is invalid`

**原因**：回调 URL 格式不正确或不可访问

**解决**：
1. 确保 URL 是完整的 HTTP/HTTPS 地址
2. 确保域名可公网访问
3. 使用内网穿透工具（如 cpolar）进行测试

## 测试代码

在 `handler/server.go` 的 `AliPay` 方法中添加详细日志：

```go
func (s *Server) AliPay(_ context.Context, in *__.PayReq) (*__.PayResp, error) {
    log.Printf("[AliPay] 收到支付请求：Id=%d", in.Id)
    
    var order model.Order
    id, err := order.FindId(config.DB, in.Id)
    if err != nil {
        log.Printf("[AliPay] 查询订单失败：%v", err)
        return nil, errors.New("查询失败")
    }
    log.Printf("[AliPay] 订单查询成功：OrderCode=%s, Price=%.2f", id.OrderCode, id.Price)
    
    pay := pkg.AliPay(id.OrderCode, id.Price)
    
    if pay == "" {
        log.Println("[AliPay] 支付 URL 为空")
        return &__.PayResp{
            Url: "",
        }, nil
    }
    
    log.Printf("[AliPay] 支付 URL 生成成功：%s", pay)
    return &__.PayResp{
        Url: pay,
    }, nil
}
```
