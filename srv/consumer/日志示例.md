## 日志级别说明

| 符号 | 说明 |
|------|------|
| ✅ | 操作成功 |
| ❌ | 操作失败 |
| 🎉 | 流程完成 |
| ℹ️ | 信息提示 |

## 关键字段说明

### 订单消息日志
- `Key`: 消息唯一标识（MD5 Hash），用于幂等性检查
- `OrderCode`: 订单编号
- `GoodsId`: 商品 ID
- `Num`: 购买数量
- `Price`: 订单总价
- `耗时`: 从开始处理到完成的总时间

### 库存扣减日志
- `原库存`: 扣减前的库存数量
- `扣减`: 本次扣减的数量
- `现库存`: 扣减后的库存数量

## 异常处理

### 1. 数据库连接失败
```
[Order] ❌ 创建订单失败：dial tcp 127.0.0.1:3306: connect: connection refused
```

### 2. Redis 连接失败（幂等性检查失败）
```
[Order] ❌ 幂等性检查失败：redis: client is closed
```

### 3. 消息解析失败
```
[Order] ❌ 解析订单消息失败：unexpected end of JSON input
```

## 监控指标

建议配置以下监控：
1. 订单处理成功率（目标：> 99%）
2. 平均处理耗时（目标：< 200ms）
3. 库存扣减失败次数（目标：0）
4. 重复消息跳过次数（用于检测重复消费）
5. Redis 连接状态
6. RabbitMQ 队列积压数量
