# 消费者日志配置说明

## 日志功能

消费者服务现在支持**同时输出日志到控制台和文件**。

## 日志文件位置

```
srv/consumer/logs/consumer_YYYY-MM-DD.log
```

**示例**：
```
srv/consumer/logs/consumer_2026-03-14.log
```

## 日志目录结构

```
srv/consumer/
├── main.go           # 消费者主程序
├── logs/             # 日志目录（自动创建）
│   ├── consumer_2026-03-14.log
│   ├── consumer_2026-03-15.log
│   └── ...
└── 日志说明.md
```

## 日志格式

```
2026/03/14 10:05:30 main.go:123 [Order] 开始处理订单消息：Key=a1b2c3d4e5f6g7h8
2026/03/14 10:05:30 main.go:125 [Order] 幂等性检查通过：Key=a1b2c3d4e5f6g7h8
2026/03/14 10:05:30 main.go:140 [Order] 订单详情：OrderCode=abc123, GoodsId=1, Num=2, Price=199.98
```

**格式说明**：
- `日期 时间`: 日志时间戳
- `文件名：行号`: 代码位置
- `[模块]`: 日志模块标识
- `消息内容`: 详细日志信息

## 日志级别

| 前缀 | 说明 |
|------|------|
| `[Order]` | 订单处理相关日志 |
| `[Consumer]` | 消费者启动/停止日志 |
| `[DB]` | 数据库连接日志 |
| `[Redis]` | Redis 连接日志 |
| `[Idempotent]` | 幂等性检查日志 |
| `[RabbitMQ]` | RabbitMQ 连接日志 |

## 日志状态标识

| Emoji | 说明 |
|-------|------|
| ✅ | 操作成功 |
| ❌ | 操作失败 |
| 🎉 | 流程完成 |

## 日志管理

### 日志文件按日期分割

- 每天生成一个新的日志文件
- 文件名格式：`consumer_YYYY-MM-DD.log`
- 自动创建，无需手动干预

### 日志清理建议

建议定期清理旧日志文件（保留最近 7 天）：

**Linux/Mac**:
```bash
# 删除 7 天前的日志
find srv/consumer/logs -name "consumer_*.log" -mtime +7 -delete
```

**Windows (PowerShell)**:
```powershell
# 删除 7 天前的日志
Get-ChildItem srv/consumer/logs/consumer_*.log | Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-7) } | Remove-Item
```

## 日志查看命令

### 实时查看日志（Linux/Mac）
```bash
tail -f logs/consumer_$(date +%Y-%m-%d).log
```

### 实时查看日志（Windows PowerShell）
```powershell
Get-Content logs/consumer_$(Get-Date -Format 'yyyy-MM-dd').log -Wait -Tail 50
```

### 搜索特定订单日志
```bash
# Linux/Mac
grep "OrderCode=abc123" logs/consumer_2026-03-14.log

# Windows PowerShell
Select-String -Path "logs/consumer_2026-03-14.log" -Pattern "OrderCode=abc123"
```

### 统计错误日志
```bash
# Linux/Mac
grep "❌" logs/consumer_2026-03-14.log | wc -l

# Windows PowerShell
(Get-Content "logs/consumer_2026-03-14.log" | Select-String "❌").Count
```

## 日志配置自定义

### 修改日志目录

编辑 `main.go` 中的 `initLogger()` 函数：

```go
func initLogger() error {
    // 修改日志目录路径
    logDir := "./logs"  // 改为其他路径，如 "/var/log/consumer"
    
    // ...
}
```

### 修改日志文件格式

编辑 `main.go` 中的日志文件名生成逻辑：

```go
// 按小时分割
logFileName := filepath.Join(logDir, fmt.Sprintf("consumer_%s.log", 
    time.Now().Format("2006-01-02-15")))

// 按月分割
logFileName := filepath.Join(logDir, fmt.Sprintf("consumer_%s.log", 
    time.Now().Format("2006-01")))
```

### 只输出到文件（不输出到控制台）

编辑 `main.go` 中的 `initLogger()` 函数：

```go
// 只输出到文件
log.SetOutput(logFile)

// 同时输出到控制台和文件（默认）
mw := io.MultiWriter(os.Stdout, logFile)
log.SetOutput(mw)
```

## 常见问题

### Q1: 日志文件在哪里？
**A**: 在 `srv/consumer/logs/` 目录下，文件名包含当前日期。

### Q2: 为什么日志文件没有创建？
**A**: 检查程序是否正常启动，日志目录是否有写权限。

### Q3: 如何查看历史日志？
**A**: 直接查看 `logs/` 目录下的历史文件，按日期命名。

### Q4: 日志文件太大怎么办？
**A**: 建议定期清理旧日志，或配置日志轮转（log rotation）。

### Q5: 如何只查看错误日志？
**A**: 使用 `grep "❌"` 或 `Select-String "❌"` 搜索错误标识。

## 日志监控建议

建议配置以下监控：

1. **日志文件大小监控**: 超过 100MB 告警
2. **错误日志数量监控**: 每小时错误超过 10 次告警
3. **日志写入延迟监控**: 最后写入时间超过 5 分钟告警
4. **磁盘空间监控**: 日志目录所在磁盘使用率超过 80% 告警
